Comparison library (C++20)

The <compare> library was introduced in C++20 to provide a standardized way to perform comparisons between objects. It introduces the concept of three-way comparison and provides tools for generating comparison operators automatically.

Key benefits of using <compare>

  • Simplified code: The library provides a way to automatically generate comparison operators, reducing the amount of boilerplate code you need to write.
  • Improved consistency: By using the library's tools, you can ensure that your comparison operators are consistent and correct.
  • Support for three-way comparison: The library introduces the concept of three-way comparison, which can provide more information about the relationship between two objects than traditional two-way comparisons.

Three-way comparison

The <compare> library introduces the <=> operator, also known as the spaceship operator, which performs a three-way comparison between two objects. The result of this operator is a std::strong_ordering, std::weak_ordering, or std::partial_ordering object, which indicates the relationship between the two objects.

Generating comparison operators

The <compare> library provides tools for automatically generating comparison operators based on the <=> operator. This can significantly reduce the amount of code you need to write to support comparisons in your classes.

Example: Using <compare> with a custom class

#include <iostream>
#include <compare>

struct MyClass {
    int value;

    auto operator<=>(const MyClass& other) const {
        return value <=> other.value;
    }
};

int main() {
    MyClass a{1};
    MyClass b{2};

    if (a < b) {
        std::cout << "a is less than b" << std::endl;
    }

    if (a == b) {
        std::cout << "a is equal to b" << std::endl;
    }

    if (a > b) {
        std::cout << "a is greater than b" << std::endl;
    }

    return 0;
}

Ordering categories

The <compare> library defines three ordering categories:

  • std::strong_ordering: Represents a total order, where all elements are comparable.
  • std::weak_ordering: Represents a partial order, where some elements may not be comparable.
  • std::partial_ordering: Represents a partial order, where some elements may not be comparable.

The choice of ordering category depends on the specific requirements of your class and the nature of the data it represents.
