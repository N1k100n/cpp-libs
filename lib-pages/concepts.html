Concepts library (C++20)

The <concepts> library was introduced in C++20 to provide a way to define constraints on template parameters. Concepts are predicates that are evaluated at compile time to determine if a given type satisfies the requirements of a template.

Key benefits of using concepts

  • Improved error messages: When a template instantiation fails due to unmet constraints, the compiler can provide more specific and informative error messages, pointing directly to the violated concept.
  • Simplified template code: Concepts make template code easier to read and understand by clearly specifying the requirements for template parameters.
  • Enabling generic programming: Concepts facilitate generic programming by allowing developers to write code that works with any type that satisfies a given set of requirements.

Defining concepts

Concepts are defined using the concept keyword followed by the concept name and a list of requirements.

template <typename T>
concept Integral = std::is_integral_v<T>;

template <typename T>
concept SignedIntegral = Integral<T> && std::is_signed_v<T>;

Using concepts in templates

Concepts can be used to constrain template parameters by placing them after the parameter name in the template declaration.

template <Integral T>
T add(T a, T b) {
    return a + b;
}

Standard library concepts

The C++ standard library provides a number of predefined concepts that can be used to constrain template parameters. Some examples include:

  • std::integral
  • std::floating_point
  • std::copyable
  • std::movable

Example: Using concepts with std::vector

#include <iostream>
#include <vector>
#include <concepts>

template <std::integral T>
void print_vector(const std::vector<T>& v) {
    for (const auto& x : v) {
        std::cout << x << " ";
    }
    std::cout << std::endl;
}

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    print_vector(v); // Output: 1 2 3 4 5

    // std::vector<double> v2 = {1.1, 2.2, 3.3};
    // print_vector(v2); // Error: Constraints not satisfied

    return 0;
}

Concepts provide a powerful mechanism for improving the type safety and expressiveness of template code in C++. They enable developers to write more generic and reusable code while ensuring that template parameters meet the required constraints.
