Functional library

The <functional> header provides a number of function objects (functors) and related tools that support functional programming in C++. Function objects are objects that can be called like functions, and they are often used to pass behavior as arguments to algorithms or other functions.

Key components in <functional>

  • Function objects: Predefined function objects for common operations like arithmetic, comparison, and logical operations (e.g., std::plus, std::less, std::logical_and).
  • Function object adapters: Tools for creating new function objects by combining or modifying existing ones (e.g., std::bind, std::not1, std::mem_fn).
  • std::function: A class template that can wrap any callable object (function, lambda expression, functor, etc.) with a specific signature.

How to use <functional>

To use <functional>, you need to include the <functional> header file. You can then use the function objects, adapters, and std::function class template in your code.

Example: Using predefined function objects

#include <iostream>
#include <functional>
#include <algorithm>
#include <vector>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    std::transform(v.begin(), v.end(), v.begin(), std::negate<int>()); // Negate all elements

    for (int x : v) {
        std::cout << x << " ";
    }
    std::cout << std::endl; // Output: -1 -2 -3 -4 -5

    return 0;
}

Example: Using std::function

#include <iostream>
#include <functional>

int add(int a, int b) {
    return a + b;
}

int main() {
    std::function<int(int, int)> f = add; // Wrap the add function
    int result = f(2, 3); // Call the function through the std::function object

    std::cout << "Result: " << result << std::endl; // Output: Result: 5

    return 0;
}

When to use <functional>

<functional> is useful when you need to work with function objects, especially in the context of generic programming and algorithms. It provides a way to abstract over different callable objects and to manipulate them in a uniform way.

However, it's important to be aware of potential performance overhead when using function objects and std::function, as they can sometimes be less efficient than direct function calls. If performance is critical, consider using templates or other techniques to avoid the overhead.
